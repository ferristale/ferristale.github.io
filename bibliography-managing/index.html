文献管理软件的选择

2018年3月15日
 | 瞎搞
文章目錄

endnote/mendeley/Citavi/zotero

作为一名科研狗，文献管理软件算是必备工具了 同时作为一个完美主义的强迫症者，我也花了很多时间在追求“最好”上面 （但实际上这并不可取）

NoteExpress

本科，我用noteexpress 因为学校买了，而且那时候中文文献看的比较多 noteexpress对中文支持差不多是最好了吧

Endnote

但到了研究生，一来没授权了，二来中文也不够用了，开始装了endnote 学校提供的是X8 对于endnote，我用得不多 印象就是

  1. 功能强大
  2. 自动下载
  3. 笔记捉急
  4. PDF阅读太卡
  5. 占用CPU

促使我换掉他的是第五点 每次打开他都占我很多的CPU，具体是什么情况我也忘了，反正那段时间对他特别不满意 有时候特别卡，就换了

Mendeley

  1. PDF搜索很强
  2. 提取元信息能力强
  3. 同步慢
  4. 文件监控有点弱？

针对各点说一说

  1. 用他的主因，那时候要统计文献，据说他在这方面很强
  2. 可以直接把文献拖进去，自动识别并改名，很爽
  3. 这个，好像说这个功能在国内不好使，推荐关闭
  4. 促使我放弃的原因

我是Win+Ubuntu双系统 两个系统均装了Mendeley并打开了文件夹监控 监控对象是我存放文献的文件夹 本来在单系统下，这样一来，我只需要把文献下下来，觉得有用，就扔进专门放文献的地方 让软件自动识别并改名，美滋滋 但是，问题来了

  1. 有些文献识别是错误的，当我切到另一个系统，他会全部重新识别，造成大量工作量
  2. 同步功能也不知道怎么地，每次打开都要同步很久，同步项目远多于文献数，且会卡在某个点
  3. 还有一点是他从文献识别的信息，和读取DOI的信息有时候格式不同，折腾多几次之后就很乱

受不了，于是就想着换别的 现在想来，其实不监控文件夹也是可以避免误会的，不过我也懒得去研究了 而同步的问题，我不切换系统也会出现，特烦人

Citavi

知乎上了解到的 据说用了你会爱上他 据说在德国使用很广 使用了一下，感受如下

  1. 功能确实别致，不同于文献管理，更偏重文献应用
  2. 但，对于13.3的小屏幕来说，装不下这么大的世界。。一看东西太多太密我就受不了

zotero

暂时用的是这个 也花了很多时间在上面 因为是开源的，有很强的定义性 用来管理文献，配合latex还是不错的 不过也有一些不如意的地方

  1. 备份方面较为复杂
  2. 文献的存放是一个个独立文件夹，文件夹名称还是独立的字符，强迫症有点受不了
  3. 具体怎么流水工作我还没明白==

这里提供两个备份方案 前置知识：

  1. 备份的结果是一个一个的文件夹，文献压缩存放，名称无意义
  2. 本地另有一个不压缩的文献库，也是一个一个的文件夹，区别的文献不压缩
  3. 本地文献库位于本地数据库中，库包含了几乎所有软件配置，可随意移动
  4. 即，你可以把数据库放到同步盘中，实现备份。缺点是会大一些。这个“备份”区别于软件定义的备份，姑且称为同步

坚果云WEBDAV备份

这是我目前使用的 在坚果云设置一个同步文件夹用来备份（最好不带中文），不同步到本地 这样，云端有一个备份，本地有一个数据库。

box网盘WEBDAV,数据库坚果云同步

结果是： 在box（或其他支持webDAV的网盘）中有一份备份 同时数据库在坚果云中有一份“备份”，实现跨平台及双重保障，还可以保障其他数据不丢失 我不选择是考虑到没必要随身携带整个数据库，需要时按需下载就可以（备份的文献可以选择按需下载），也算一种节约硬盘的方式吧

因为现在支持自定义数据库目录了，所以以前软连接的方式其实没有必要 不过这里也是针对我目前使用情况的方案，以后会用什么方案，会用什么软件，还不好说，也许我会回到Citavi呢？ 而看网上文章，好像还有其他很多考量，包括结合插件的玩法，我还没去折腾，以后再说

今天跟同学出去吃个饭，还坐了会咖啡厅，回来时间都不怎么够，为了写完这个写到12点40，肚子都饿了。我还立了flag要早睡的
