終於用上了Hexo

Apr 27, 2016 | 瞎搞

終於用上Hexo了

如果你有看我的關於就會知道，前不久我才換了一個新主題，這麼快就連框架也換了

是的沒錯不僅框架換了，事實上服務器也“翻新”了：P

選來選去選了這個，Hexo

到現在，博客也算經歷了幾大更迭

  1. Namecheap的廉價主機自帶的WP
  2. 搬瓦工VPS加VESTACP
  3. VPS直接上一鍵安裝包+WP
  4. Nginx+Hexo

相比之前，服務器閒了好多，然而這個過程卻是相當折騰==

昨晚，就是昨晚，WIN上本地編譯，成功渲染出靜態版的博客

於是激動啊，想都沒想就把服務器reload了

結果……

img

在经过15个小时的Down机之后，终于是把文件传上去了

至于为什么一直传不上，我也不知道具体原因

可以肯定的是跟win有关

因为我中午的时候装好Ubuntu，一次成功==

我还能说什么0.0

当然啦这里也是有些小插曲的

我选择的是同时deploy到coding和VPS

  • git的话，第一次用，搞不懂怎么pull，生成了无数遍KEY之后
    才算成功
  • rsync 這貨纔是真正的棘手，照着網上的教程試了好多邊
    可是呢，那些教程要麼不全，要麼抄襲，整一個忽悠
    比如說本地和服務器都要安裝，比如說服務器還要裝xinet
    比如說服務器裝完rsync要改配置
    比如說服務器啓動rsync有兩種方法
    比如說Hexo的插件似乎並非支持全部方法

心好累，無言以對，這些都是我花了相當多時間才搞清楚

最後，本地和服務器終於SSH通

卻是Connect failed

心塞==

更心塞的是Hexo的issue，四天都沒人回

一氣之下，重新裝上Ubuntu
（恢復備份的時候說Hash對不上！我真的是。。。）

部署環境，hexo d

oh…..!!!!!

居然成功了！

我還能說什麼

第一次用Markdown

是的你沒有看錯，這是我第一次用Markdown

是的沒錯我不是因爲Markdown才轉入Hexo的 PiaPiaPia

其實早就對Markdown有所耳聞啦，不過一直沒有實質性的進展

這次轉投Hexo，算是逼迫自己學了一下MD

不得不說真的是心曠神怡！從未有過的舒暢！

終於明白爲什麼MD備受推崇了！！！

另外不得不說的是，關於編輯器，知乎上的一個回答讓我深受啓發:

Markdown 这种以适合直观阅读为特色的简单标记文本格式，目的是无障碍的直接阅读，不被编辑器功能限制，而能输出简单的格式，为什么要纠结于如何选择编辑工具，还总是强调写作工具的预览功能？
如果用户觉得 Markdown 格式文本不够直观，还要实时预览才放心安心，还这么挑剔编辑器，那又何必用 Markdown 写作呢？这不是违背初衷吗？
用带 HTML 快捷按键的文本编辑器加浏览器预览的选择会多得多。

於是，我吊絲地使用了自帶的gedit（不過作爲一個新手，特別是第一次的新手，還是要偶爾複製到在線編輯器看看對不對==）
（不過話說回來你們幾個語法有點出入是幾個意思！！）

一點番外

今天分手了
今天去見了實驗導師，感慨萬千，總之，真正的大學生活要開始了
手機爆屏了。。還好買了保險不然就是買買賣了不過講真我忍這手機很久了！！
